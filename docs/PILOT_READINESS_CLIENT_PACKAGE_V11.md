# GeoAI Pilot Readiness & Client Delivery Package v1.1

Date: 2026-06-21

GeoAI v1.1 introduces a practical client pilot framing layer for the existing MVP. It does not make the product production-ready or decision-authoritative. It clarifies how the current demo can be converted into a controlled 2-4 week client pilot using uploaded/customer-approved data, open snapshots and agreed validation sources.

## Purpose

The pilot readiness package helps a client and GeoAI agree:

- which pilot package fits the client type;
- what input data is required;
- which outputs will be delivered;
- which evidence sources are demo/open/customer-provided;
- which official or customer-approved validation steps remain before decisions.

## Pilot Packages

### Developer / Master Developer Site Screening Pilot

Objective: screen 3-10 candidate land parcels or development areas for investment, planning and infrastructure readiness.

Typical deliverables:

- site screening dashboard;
- ranked opportunity and risk summary;
- development due diligence memo;
- comparison matrix for shortlisted sites;
- validation checklist for official follow-up.

### Fund / Family Office Investment Screening Pilot

Objective: compare investable Dubai locations or assets using market context, spatial evidence and risk-adjusted recommendation logic.

Typical deliverables:

- investment screening dashboard;
- risk-adjusted comparison table;
- investment memo preview;
- evidence and data confidence summary;
- next due diligence action list.

### Bank / Lender Asset Review Pilot

Objective: review collateral or financed asset locations for spatial risk, market context, evidence gaps and lender-ready reporting.

Typical deliverables:

- asset review dashboard;
- collateral context summary;
- risk and validation gap memo;
- shortlist comparison for priority assets;
- evidence lineage summary.

### Government / Free Zone Land Monitoring Pilot

Objective: monitor selected land, district or free zone assets with transparent evidence lineage, change context and validation workflow.

Typical deliverables:

- land monitoring dashboard;
- selected object evidence summary;
- planning and infrastructure context memo;
- comparison view for monitored assets;
- validation and escalation checklist.

## Client Data Required

Each pilot requires a small client-approved dataset. Depending on client type, this usually includes:

- target sites, assets or monitored areas;
- coordinates, addresses or GeoJSON boundaries;
- client objective or investment/development thesis;
- required scenario and risk dimensions;
- known validation sources and internal approval requirements.

Recommended but optional inputs include internal feasibility notes, underwriting assumptions, inspection records, project status snapshots, comparable evidence, or existing memo templates.

## External / Open Data Context

Current MVP context can include synthetic demo layers, open-geospatial baseline fixtures, uploaded client files, and sample/offline market snapshots. These sources help demonstrate workflow, evidence lineage and report output.

They do not replace official validation. GeoAI should not claim live DLD, live Dubai Pulse, GeoDubai, official parcel, zoning, cadastral, ownership, certified valuation or regulatory approval coverage until those integrations and permissions are explicitly in place.

## Pilot Timeline

### Week 0 — Scoping

- Select client type and pilot package.
- Confirm 3-10 target sites/assets.
- Agree data fields and validation owners.
- Define success criteria.

### Week 1 — Data Setup

- Load customer-approved CSV/GeoJSON or asset metadata.
- Confirm open/demo source status.
- Configure project workspace and scenario defaults.
- Identify official validation sources.

### Week 2 — Analysis Workflow

- Run Express Analysis for target assets.
- Compare shortlisted sites/assets.
- Review evidence cards, limitations and validation gaps.
- Refine client memo structure.

### Week 3 — Report Package

- Generate pilot memos and comparison reports.
- Prepare validation checklist and next actions.
- Review outputs with client stakeholders.

### Week 4 — Decision On Next Phase

- Evaluate workflow fit, data readiness and evidence confidence.
- Decide whether to connect official/customer production sources.
- Scope security, auth, persistence and governance requirements.

## Success Criteria

A successful pilot should show that:

- the client can screen or compare priority sites/assets in one workflow;
- each result has visible source lineage and data-confidence language;
- the memo/report output is useful for due diligence conversations;
- validation gaps are explicit and actionable;
- stakeholders can decide which production integrations are worth building next.

## Limitations

GeoAI v1.1 remains a demo/pilot-readiness MVP. It is not production-ready, not a certified valuation tool, not a regulatory decision system, and not a substitute for official planning, title, ownership, cadastral, zoning, market or legal validation.

## Commercial Pilot Framing

The recommended commercial framing is a time-boxed 2-4 week pilot with:

- a defined client type and pilot package;
- 3-10 target sites/assets;
- uploaded/customer-approved data;
- open or sample context clearly labeled;
- agreed validation sources;
- a dashboard, comparison view and memo/report deliverable;
- a final review of data gaps, workflow value and production requirements.
