# GeoAI Brand Foundation v1

Status: active brand foundation / implementation blocked  
Date: 2026-07-07  
Branch: `design-audit-v1`

## Executive summary

This document defines the first structured brand foundation for GeoAI Light.

The goal is to keep the current product UX geometry intact and redesign only the visual layer: color, typography, density, component emphasis, map styling, source states and evidence/caveat treatment.

## Brand direction

GeoAI should feel:

- premium;
- calm;
- precise;
- evidence-led;
- international;
- enterprise SaaS;
- spatially intelligent;
- not generic GIS;
- not colorful dashboard software.

## Brand principles

1. Calm intelligence: quiet confidence, no visual noise.
2. Evidence before decoration: colors support source state and decisions.
3. Spatial clarity: the map remains the decision context, not background art.
4. Premium restraint: few colors, precise spacing and strong hierarchy.
5. Official validation honesty: no legal, cadastral, zoning, planning or valuation claims.

- Validation Amber: `#F59E0B`

- Red Soft: `#FEECEC`

## Color usage ratio

- 75% neutrals;
- 15% Geo Teal;
- 8% Signal Blue;
- 2% validation amber or critical semantic states.

## Typography

Primary typeface: Inter.

Rules:

- wide title boxes;
- no unnecessary heading wrapping where horizontal space exists;
- subtitles must start below the title zone;
- 120-130% line-height;
- dense enterprise UI uses 12-16px text only where readable.

Recommended scale:

- Display: 52 Bold;
- H1: 40 Bold;
- H2: 28 Bold;
- Section: 18 Semi Bold;
- Body: 14 Regular;
- Meta: 12 Semi Bold.

## Spacing and shape

Use an 8pt grid.

Spacing scale:

- 4, 8, 12, 16, 20, 24, 32, 40, 48.

Radius scale:

- fields: 8px;
- compact cards: 12px;
- standard cards: 16px;
- major surfaces: 20-24px.

## Map visual system

The map should stay light, precise and decision-oriented.

Map roles:

- water / coast context: light blue;
- land / urban base: warm off-white;
- selected AOI / object: Signal Blue;
- open context layer: Geo Teal;
- validation gap overlay: Validation Amber;
- no official claim state: neutral surface.

## Component application rules

- Right AnalysisPanel uses Geo Teal for active mode, Signal Blue for primary CTA and Amber for validation caveats.
- Project/readiness tables use neutral surfaces, compact rows and explicit next validation step.
- Cards must contain context, evidence, decision, action or output.
- Decorative empty cards are not allowed.
- All titles get full available width.
- Before editing a current version, duplicate it and move the duplicate left as the previous version.

## Required caveat

“Screening hypothesis; official validation required; not a legal, cadastral, zoning, planning or valuation conclusion.”
