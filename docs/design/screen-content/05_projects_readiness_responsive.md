# Projects, Readiness and Responsive Content Spec

Status: draft.

## Projects

The Projects screen needs:

- project overview
- saved AOIs
- analysis history
- report packages
- action priorities

Design note: project content must be project-scoped only.

## Readiness

The Readiness screen needs:

- backend status
- storage status
- source status
- audit status
- known limitations
- pilot gate checklist

Design note: readiness must be separate from the first user decision flow.

## Responsive states

Required behavior:

- large desktop: full product shell
- compact desktop: reduced rail and panel density
- tablet: map first, cards below
- mobile: top bar, section tabs, context, card, action

Design note: mobile must not be a squeezed desktop dashboard.
