# GeoAI Data License And Caveats v1.6

Date: 2026-06-23

GeoAI v1.6 keeps public data usage in a screening, snapshot and source-lineage mode.

Required caveat:

> screening hypothesis; official validation required; not a legal, cadastral, zoning, planning or valuation conclusion.

## License Caveats

- DLD / Dubai Pulse public files may have dataset-specific terms, attribution requirements and redistribution limits.
- OSM / Geofabrik context requires ODbL compliance and attribution handling.
- Overture Maps data requires Overture license and attribution review.
- Open-Meteo, NASA POWER and OpenAQ require source attribution and API terms review.
- WorldPop requires dataset-specific citation and redistribution review.
- GADM has non-commercial/permission caveats and must not be used as a default commercial source.
- Copernicus / Sentinel metadata and imagery require mission/product-specific review.

## Data Honesty Rules

Allowed wording:
- public snapshot
- open snapshot
- API context
- sample fallback
- manual import ready
- permission required
- planned official validation

Forbidden wording:
- live DLD integration
- live Dubai Pulse integration
- live GeoDubai GIS
- official parcel boundary
- official zoning boundary
- cadastral validation
- ownership verification
- certified valuation
- certified flood risk
- engineering-grade climate assessment
- insurance-grade hazard model
- officially suitable
- approved
- guaranteed best use

## Validation Requirements

Before client or operational decisions, validate:
- ownership, title and parcel information through authorized sources;
- zoning, planning and permitted use through authorized municipal/customer-approved sources;
- valuation and investment conclusions through qualified professional processes;
- climate, flood, energy and insurance implications through engineering or specialist review;
- dataset licenses, attribution and permitted commercial use.
